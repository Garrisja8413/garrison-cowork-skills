# Numbering Definitions Specification — Shared Reference

This is the single source of truth for all Word numbered list definitions
used across PA-DOC-FORMAT overlay skills. **All numbered items MUST use
Word's built-in numbered list mechanism via `<w:numPr>`, NOT inline text
numbers.**

## Heading Numbering (abstractNumId="5", numId="6")

The heading numbering scheme uses a multilevel list definition. This MUST
be included in the numbering.xml part of any generated document that uses
Heading 1-6 styles.

**CRITICAL:** The heading text MUST NOT contain the number. The number
is auto-generated by this definition. For example, a Heading 1 paragraph
with text "LEGAL STANDARD" will render as "I. LEGAL STANDARD" because
the numbering definition produces the "I." prefix.

```xml
<w:numbering xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main">
  <w:abstractNum w:abstractNumId="5">
    <w:multiLevelType w:val="multilevel"/>
    <w:tmpl w:val="PA0HEADS"/>
    <!-- Level 0: I., II., III. (Heading 1) -->
    <w:lvl w:ilvl="0">
      <w:start w:val="1"/>
      <w:numFmt w:val="upperRoman"/>
      <w:pStyle w:val="Heading1"/>
      <w:lvlText w:val="%1."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="720" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Level 1: A., B., C. (Heading 2) -->
    <w:lvl w:ilvl="1">
      <w:start w:val="1"/>
      <w:numFmt w:val="upperLetter"/>
      <w:pStyle w:val="Heading2"/>
      <w:lvlText w:val="%2."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="1440" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Level 2: 1., 2., 3. (Heading 3) -->
    <w:lvl w:ilvl="2">
      <w:start w:val="1"/>
      <w:numFmt w:val="decimal"/>
      <w:pStyle w:val="Heading3"/>
      <w:lvlText w:val="%3."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="2160" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Level 3: a., b., c. (Heading 4) -->
    <w:lvl w:ilvl="3">
      <w:start w:val="1"/>
      <w:numFmt w:val="lowerLetter"/>
      <w:pStyle w:val="Heading4"/>
      <w:lvlText w:val="%4."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="2880" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Level 4: (i), (ii), (iii) (Heading 5, optional) -->
    <w:lvl w:ilvl="4">
      <w:start w:val="1"/>
      <w:numFmt w:val="lowerRoman"/>
      <w:pStyle w:val="Heading5"/>
      <w:lvlText w:val="(%5)"/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="3600" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Level 5: (a), (b), (c) (Heading 6, optional) -->
    <w:lvl w:ilvl="5">
      <w:start w:val="1"/>
      <w:numFmt w:val="lowerLetter"/>
      <w:pStyle w:val="Heading6"/>
      <w:lvlText w:val="(%6)"/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="4320" w:hanging="720"/>
      </w:pPr>
      <w:rPr>
        <w:rFonts w:hint="default"/>
      </w:rPr>
    </w:lvl>
    <!-- Levels 6-8: reserved, decimal fallback -->
    <w:lvl w:ilvl="6">
      <w:start w:val="1"/>
      <w:numFmt w:val="decimal"/>
      <w:lvlText w:val="%7."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="5040" w:hanging="720"/>
      </w:pPr>
    </w:lvl>
    <w:lvl w:ilvl="7">
      <w:start w:val="1"/>
      <w:numFmt w:val="lowerLetter"/>
      <w:lvlText w:val="%8."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="5760" w:hanging="720"/>
      </w:pPr>
    </w:lvl>
    <w:lvl w:ilvl="8">
      <w:start w:val="1"/>
      <w:numFmt w:val="lowerRoman"/>
      <w:lvlText w:val="%9."/>
      <w:lvlJc w:val="left"/>
      <w:pPr>
        <w:ind w:left="6480" w:hanging="720"/>
      </w:pPr>
    </w:lvl>
  </w:abstractNum>

  <!-- Concrete numbering instance -->
  <w:num w:numId="6">
    <w:abstractNumId w:val="5"/>
  </w:num>
</w:numbering>
```

### How Headings Reference the Numbering

Each heading style references the numbering via `<w:numPr>`:

- **Heading 1** sets `<w:numId w:val="6"/>` (ilvl defaults to 0)
- **Heading 2** sets `<w:ilvl w:val="1"/>` (inherits numId from Heading 1)
- **Heading 3** sets `<w:ilvl w:val="2"/>` (inherits numId from chain)
- **Heading 4** sets `<w:ilvl w:val="3"/>` (inherits numId from chain)
- **Heading 5** sets `<w:ilvl w:val="4"/>` and `<w:numId w:val="6"/>`
- **Heading 6** sets `<w:ilvl w:val="5"/>` (inherits numId from Heading 5)

The `<w:pStyle>` attribute on each `<w:lvl>` links the numbering level
to the heading style, so Word automatically applies the correct level
when the user applies a heading style.

### Heading Numbering Summary

| Level | Format | Example | Style | outlineLvl |
|---|---|---|---|---|
| 0 (Heading 1) | Roman upper | I. | Bold | 0 |
| 1 (Heading 2) | Alpha upper | A. | Bold | 1 |
| 2 (Heading 3) | Decimal | 1. | Bold | 2 |
| 3 (Heading 4) | Alpha lower | a. | Bold | 3 |
| 4 (Heading 5) | Roman lower | (i) | Bold | 4 |
| 5 (Heading 6) | Alpha lower | (a) | Bold | 5 |

## Complaint Paragraphs

All complaint factual allegations, causes of action paragraphs, and
damages paragraphs MUST use a Word numbered list style.

```xml
<w:abstractNum w:abstractNumId="[complaint-id]">
  <w:multiLevelType w:val="singleLevel"/>
  <w:tmpl w:val="PA0COMPL"/>
  <w:lvl w:ilvl="0">
    <w:start w:val="1"/>
    <w:numFmt w:val="decimal"/>
    <w:lvlText w:val="%1."/>
    <w:lvlJc w:val="left"/>
    <w:pPr>
      <w:ind w:left="720" w:hanging="360"/>
    </w:pPr>
  </w:lvl>
</w:abstractNum>
```

Renders as "1.", "2.", "3.", etc. automatically. Double-spaced, justified,
Optima 12pt. **Do NOT type "1." or "¶ 1." as inline text.**

```xml

  <w:multiLevelType w:val="singleLevel"/>
  <w:tmpl w:val="PA0UMF00"/>
  <w:lvl w:ilvl="0">
    <w:start w:val="1"/>
    <w:numFmt w:val="decimal"/>
    <w:lvlText w:val="UMF No. %1."/>
    <w:lvlJc w:val="left"/>
    <w:pPr>
      <w:ind w:left="1440" w:hanging="720"/>
    </w:pPr>
  </w:lvl>
</w:abstractNum>
```

Renders as "UMF No. 1.", "UMF No. 2.", etc. One fact per numbered
paragraph. Evidence cite follows each fact statement.

## Why Word Numbered Lists Matter

Attorneys frequently add, delete, and reorder items. If numbering is
inline text, every subsequent number must be manually updated — creating
errors. Word's numbering engine handles renumbering automatically.

This applies to:
- **Heading numbers** — reordering sections
- **Complaint paragraphs** — adding/removing allegations
- **UMF facts** — adding/removing material facts
- **Discovery items** — adding/removing interrogatories, RFPs, RFAs
- **Deposition topics** — adding/removing topics

## abstractNumId Assignment

When a document uses multiple numbering definitions (e.g., headings +
complaint paragraphs + UMFs), each gets a unique `abstractNumId`. The
heading definition uses `abstractNumId="5"` and `numId="6"` by
convention. Other definitions use subsequent IDs.

Example for a complaint document:
- abstractNumId="5" → Heading numbering (I./A./1./a.)
- abstractNumId="6" → Complaint paragraph numbering (1., 2., 3.)
- numId="6" → References heading abstractNum (abstractNumId="5")
- numId="7" → References complaint abstractNum (abstractNumId="6")
